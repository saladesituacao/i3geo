Connecting to WMS services

Prerequisites

To connect your ArcGIS 9.x software to WMS services you need download and install ArcGIS Service Pack 3.

Steps

  1. Navigate to the folder GIS Servers of ArcCatalog or open Add Data dialog of ArcMap. Double click on Add WMS Server.
  2. Add WMS server dialog appears. On URL input a discovery URL of your WMS Geoserver. Press Get Layers button to get a list of advertized layers.
  3. You can select the root layer of the service (second of list) or a single layer as you desire. Click OK.
  4. Once added, the service will be listed on GIS Servers category of the catalog. You can add layers from the service to your documents navigating to this item.

Connecting to WFS services (read only)

ArcGIS 9.x supports WFS through Data Interoperability Extension by Safe Software (read only).

Prerequisites

To connect your ArcGIS 9.x software to WFS services you need buy and install ArcGIS Data Interoperability Extension.

Steps

  1. Navigate to the folder Interoperability Connections of ArcCatalog .You need to set up the WFS connection from ArcCatalog before attempt to add WFS layers from ArcMap.
  2. On dialog Interoperability Connections select more formats (...) . In the Formats Gallery dialog select Web Feature service (WFS) format and click OK.

  3. Choose an appropriate Coordinate System (GCS_WGS_1984 for most WFS servers) you wish to retrieve the data.
  4. Back to ArcMap and add WFS layers to your map.

Connecting to WFS-T services (read, write & locking)

ArcGIS 9.x supports WFS Feature transactions (WFS\-T) through wfsEditor Extension by con terra making full use of the locking and transactional capabilities of GeoServer.

Prerequisites

To connect your ArcGIS 9.x software to WFS-T services you need buy and install wfsEditor Extension.

Features

  • Processing of ArcSDE datasets and Shapefile repositories even with ArcView
  • Interoperability through use of transactional OGC Web Feature Services (WFS\-T) with FeatureLocking
  • Comfortable editing of spatial objects using standard processing tools available in the ArcGIS Desktop client
  • Support of online editing and long-term transactions
  • Simplified editing strategy made possible by implementing an exclusive lock concept
  • Status information on individual objects and editing sessions available at all times
  • Ideal as an addition component to the sdi.suite for managing spatial data in an SDI
  • Simple administration
  • Extension for ArcGIS from version 8.3
